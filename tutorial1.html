Elements, tags, and attributes in HTML

An element is composed of the opening tags, closing tags, and everything that lies between those tags.

Tags are either opening or closing. They tell you more about the content that lies b/w the opening and closing tags.

Opening tags can contain attributes. They tell us more about the contents inside the element.

An attribute has a name and a value. The two are separated by an equals sign.

Example: lang="en-us" (values are encased inside double quotes)

Many attributes can only be used on certain elements. Such as only in head, body, etc. Other can appear on any element such as lang.

Most attribute values are fixed meaning that the values are already determined for that attribute and you have to choose which one to use depending on what you want.

Generally, every HTML document (webpage) will have a head and a body. The head give you information on what the content of the body will contain.

The body will contain whatever content you want to display such as new articles, images, videos, etc.

Ways to edit the structure of text in HTML files.

Structural markup: elements you use to describe headings and paragraphs

HTML has six levels of headers (h1-h6) with h1 being the largest and h6 being the smallest.

Use the "b" element to make text bold.

Use the "i" element to make text into italics.

Use the "sup" element to make text into superscript. Today is April 13th.

Use the "sub" element to make text into subscript. CO2.

Use the "br /" element to line break inside the middle of a paragraph. I am going to
try and line break now.

Use the "hr /" element to add horizontal rules or borders between sections.

The "hr /" element is an example of an empty element which is an element that does not have any words between the opening and closing tags.

An empty element usually has only one tag.

Semantic markup: elements you use to provide extra information about headings and paragraphs

Semantic markups do not change the structure of your text but simply provides more information about the text such as whether part text has a strong empahsis or is actually a quote.

What is a document type declaration?

In HTML5, empty elements are elements that are nontextual or contain directions on how a page should be formated.

For instance, images do not contain text and are considered empty elements. Break lines, "br /" are also considered empty elements.

Empty elements are entered using one-sided tag which means they are simply written using either one opening tag OR one closing tag. Usually we use the closing tag format. "element /"

HTML files always start with a document type declaration which tells the web browser which markup language to use.

Each markup language (HTML5, XHTML2.0, HTML4.01, etc.) has a different declaration.

The document type declaration for HTML5 is "!DOCTYPE html" and is encased inside angular brackets.

The doctype is followed by the html element, then the head element, and finally the body element. Each of these elements have starting and ending tags.
So the very last tag should always be the closing tag of the html element because the head and the body elements are nested inside the html element.

Writing the Document Head

The document head contains "metadata" (not to be confused with the meta tag). Metadata is information that tells the browser how a document (webpage) should be processed.

The "head" metadata element contain a collection of many metadata elements that work together to describe the document (webpage) or tells the browser to perform a certain actions.

The "meta" metadata element is used to define keywords, decribe content, and provide other information to the web browser.

The "script" metadata element is used to write Javascript or point to external script files through the src attribute.
This allows you to control certain aspects of other elements such as "img" for image manipulation in your document through the use of code

The "link" metadata empty element is used to describe the link between a document (webpage) and an external resource (image, another document, URL, language, etc.)

The "style" metadata element is used to provide information (by typing CSS code) about how an element in a webpage should be rendered by its browser.

The "title" metadata element stores the name of the browsertab or title bar.

Writing the Document Body

Section elements: divides major parts of the document into areas such as a header (opening content that introduces an article or section) , article, footer, headings, navigation bars, sections, etc.

Grouping elements: groups similar content together such as paragraphs, main content of a document (news articles), figures, quotes, etc.

Text-level elements: elements that are contained inside a piece of text such as "bold", "italicized", "cite", "strong", "em", etc.Also includes "cite" which is used to mark the author or title of a
quote or creative work used in your content and "q" which is used to mark the actual quote coming from the piece of creativework (books, magazines, websites, etc.).

Linking an HTML Document to A Style Sheet

Style sheet: set of rules that tell the web browser how to display the elements of a document

Linking a HTML document to a style sheet is done using the "link" metadata element and the href (location of linked document) and the rel (relationship b/w html document and linked document) attributes.
link href="file" rel="stylesheet"

To load a style sheet onto an HTML document, BOTH the HTML document and CSS file must be in the same directory (same folder).

Blockquote elements MUST be outside of the "p" element. There will be validation errors if you nest the blockquote inside the paragraph element.
